Advies Document

Introductie

Tijdens mijn stage als frontend developer heb ik diverse adviezen gegeven om de website te verbeteren. In dit document zal ik de belangrijkste adviezen bespreken die ik heb gegeven en hoe deze zijn opgevolgd. Hoewel niet alle adviezen zijn opgepakt, ben ik blij dat ik mijn kennis en ervaring heb kunnen delen en hopelijk op een positieve manier heb bijgedragen aan de verbetering van de website.

Design

De structuur van de website was complex en onduidelijk. Op basis van mijn advies besloten mijn stagebegeleiders om het design aan te passen. Ik stelde voor om de lay-out te verbeteren om de website visueel aantrekkelijker te maken en om de navigatie voor gebruikers te vereenvoudigen. Mijn stagebegeleiders volgden mijn advies op en de lay-out van de website werd verbeterd. Dit resulteerde in een aantrekkelijkere website met een verbeterde gebruikerservaring

Gebruik maken van Github

Tijdens mijn stage heb ik voorgesteld om Github te gebruiken voor het beheren van de code van de website, in plaats van Filezilla. Github biedt verschillende voordelen in vergelijking met Filezilla, zoals betere versiebeheer, gemakkelijker samenwerken met teamleden en een groter aantal tools voor het testen en debuggen van code. Helaas hebben mijn stagebegeleiders dit advies niet opgevolgd en hebben ze besloten om Filezilla te blijven gebruiken. Trello

Ik heb ook voorgesteld om Trello te gebruiken om de taakindeling en vooruitplanning te verbeteren. Trello is een projectbeheer tool die kan worden gebruikt om taken te plannen, te volgen en te delen met teamleden. Het kan ook worden gebruikt om deadlines en verantwoordelijkheden te bewaken. Mijn stagebegeleiders hebben mijn advies opgevolgd en konden mijn processen via Trello volgen.
